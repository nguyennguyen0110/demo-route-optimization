This page is used for demo of Route Optimization project.

NOTE:

  • This page ONLY for demo, not APIs. Do not copy these demo page addresses as APIs.
  • There is "/api" before each API below. Example: "/api/optimize-route". Read API document for more details.
  • Link to source code
  • The Google Drive folder below contains notebooks demonstrating data exploration, feature engineering and selection, as well as model training and fine-tuning.
  • Link to Google Drive

Current APIs:

  1. /optimize-route: optimize visit order to get the shortest route begin from depot (distributor) through locations (customers) and turn back to depot.
  2. /sale-route: optimize visit order to get the fastest route, support time window configuration for locations.
  3. /check-overlap: check for overlap routes.
  4. /suggest-frequency: suggest visit frequency for a customer base on their information from database, or input their (estimated) information if new.
  5. /cluster-customer: cluster customer into groups base on their position.
